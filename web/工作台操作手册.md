# SAC Studio 工作台操作手册

## 1. 手册说明

SAC Studio 是 Solution Practices 仓库的可视化工作台，用于查看正式解决方案、组织交付任务、检查质量结果、了解 Skills 与 Agent 的关系，以及盘点交付资产。

当前版本是静态交互原型：页面能够读取仓库数据、展示构建时质量报告并完成前端交互，但不会实际执行 Agent、创建云资源、上传 OBS、生成生产部署链接或完成发布审批。

适用人员：

- 业务人员：提交和确认交付需求，查看方案价值与交付进度。
- 解决方案架构师：查看区域、部署形态、资源拓扑和决策点。
- 开发与测试人员：查看计划文件、质量门禁、问题和资产清单。
- 交付人员：查看发布状态和待生成的交付物。

## 2. 访问工作台

### 2.1 当前本地预览

服务监听地址：

```text
0.0.0.0:4173
```

浏览器访问：

```text
http://<服务器 IP>:4173/
```

在服务器本机访问：

```text
http://127.0.0.1:4173/
```

### 2.2 开发模式

```bash
cd web
npm install
npm run dev
```

默认访问地址为 `http://localhost:3000/`。开发模式适合修改页面并即时查看效果。

### 2.3 静态构建与预览

在仓库根目录执行：

```bash
cd web
npm run build
cd ..
.venv-sac/bin/python -m http.server 4173 --directory web/out --bind 0.0.0.0
```

`npm run build` 会把仓库数据和质量报告固化到 `web/out/`。仓库内容变化后必须重新构建，静态页面才会更新。

## 3. 工作台信息架构

左侧导航分为三个区域：

| 导航组 | 页面 | 主要用途 |
|---|---|---|
| 主工作台 | 交付工作台 | 输入需求并生成交付任务卡 |
| 主工作台 | 交付任务 | 查看 Agent 交付流水线、决策点和计划产物 |
| 四个中心 | 解决方案库 | 查看正式方案、筛选能力并进入方案详情 |
| 四个中心 | 质量中心 | 查看真实质量门禁、问题和区域覆盖矩阵 |
| 四个中心 | Skills | 查看 Skill 元数据、依赖和 Agent 绑定关系 |
| 四个中心 | 资产管理 | 盘点 Terraform、文档、扩展配置和部署链接 |
| 记录 | 发布记录 | 查看 `release/` 中已有发布资产 |
| 记录 | 报告 | 查看现有分析和汇总报告 |

移动端点击页面右上角菜单按钮打开相同导航。

## 4. 推荐操作流程

```text
输入需求 → 检查任务卡 → 确认决策点 → 查看 Agent 流水线
→ 检查方案详情 → 查看质量门禁 → 核对资产 → 进入发布记录
```

### 4.1 输入交付需求

1. 打开“交付工作台”。
2. 在需求输入框中描述应用、区域、部署形态、依赖和文档语言。
3. 点击“生成任务卡”。
4. 检查任务卡中的应用、站点、区域、形态、数据库和文档语言。
5. 查看“计划输出”和“待确认决策”。

当前原型使用 LiteLLM HA 预设任务演示页面流程，尚未接入自然语言解析服务。修改输入文本不会生成另一套真实任务参数。

### 4.2 处理区域冲突

任务卡会突出显示区域语义冲突。当前示例中的正确映射为：

| 地域 | Region |
|---|---|
| 亚太-新加坡 | `ap-southeast-3` |
| 中国-香港 | `ap-southeast-1` |

确认区域前不要进入真实模板开发或云资源操作。

### 4.3 进入交付流水线

点击“确认并查看流水线”进入“交付任务”。页面包含五个阶段：

1. 架构阶段：可行性、资源拓扑、部署入口和风险。
2. 开发阶段：计划生成的 Terraform、文档和所用 Skills。
3. 测试与安全阶段：Terraform、目录、脚本、网络、凭证和供应链检查。
4. 文档阶段：中英文部署指南、方案详情和一致性检查。
5. 交付阶段：交付包、校验和、部署链接和发布版本。

点击任一阶段可查看阶段详情。右侧会展示输入、输出、决策点、Agent 日志和风险建议。

“确认决策（演示）”目前不写入数据库，也不会启动 Agent；刷新页面后选择状态会恢复。

## 5. 流水线状态说明

| 状态 | 含义 | 建议操作 |
|---|---|---|
| 未开始 | 上游条件尚未满足 | 先完成前序阶段 |
| 运行中 | Agent 正在处理该阶段 | 查看当前输出和日志 |
| 等待用户确认 | 存在会改变范围、成本或安全边界的决策 | 用户确认后才能继续 |
| 已通过 | 阶段检查通过 | 进入下一阶段 |
| 未通过 | 存在阻断问题 | 修复后重新检查 |

“等待用户确认”优先级高于普通进度提示，不能当作普通警告跳过。

## 6. 解决方案库

### 6.1 查找方案

解决方案库只展示 `project.config.json` 声明的正式方案：

- LiteLLM
- Supabase
- openJiuwen

可以按以下条件筛选：

- 方案名称、类型或区域关键词
- CN / INTL 站点
- Standard / HA 部署形态
- 中文 / English 文档
- 一键部署能力

### 6.2 方案卡操作

- “查看详情”：进入六 Tab 方案详情页。
- “复制方案”：当前为交互占位，尚未创建新的任务副本。
- “开始部署”：返回交付工作台，通过任务卡进入流水线。

### 6.3 方案详情页

方案详情分为六个 Tab：

1. 方案概览：业务价值、支持形态、覆盖区域和部署条件。
2. 架构拓扑：查看 Standard / HA 结构化资源拓扑。
3. 部署配置：查看区域、数据库、网络和待确认配置。
4. 代码：选择并预览方案目录中的真实 Terraform 脚本。
5. 质量报告：查看该方案的检查状态。
6. 文档与交付：查看中英文文档、部署链接和发布版本状态。

LiteLLM HA 拓扑主链路为：

```text
Internet → ELB → CCE → LiteLLM Pods
                         ├→ RDS PostgreSQL
                         ├→ DCS Redis
                         └→ NAT → 外部模型 API
```

当前拓扑节点是结构化演示组件，尚未实现点击跳转到 Terraform 行号。

## 7. 质量中心

质量中心直接运行 SAC 测试运行器并消费 JSON 结果。当前页面包含：

- 总质量门禁状态
- 检查实例、检查项、错误和警告数量
- 六类门禁：目录规范、Terraform、脚本质量、文档完整性、网络与安全、RFS 策略
- 可按全部、错误、警告筛选的问题列表
- 正式方案区域覆盖矩阵

点击问题实例可以展开具体检查名称和说明。

质量数据是构建时快照。代码或 practice 变化后执行：

```bash
cd web
npm run build
```

质量运行器返回非零状态时，页面仍会读取并展示错误报告；非零状态不再被误判为“0 个实例”。

## 8. Skills

Skills 页面读取仓库根目录的 `skills-index.json`，展示：

- Skill 名称、类别和状态
- Token 预算
- 关联 Agent 数量
- 前置依赖
- Skill 简介
- Agent 的基础、主要和可选 Skill 绑定

该页面属于高级模式。它用于解释 Agent 为什么具备某项能力，不是业务方案评估页，也不会直接执行 Skill。

## 9. 资产管理

资产管理页面只读扫描正式 practice，统计：

- Terraform：`.tf` 和 `.tf.json`
- Markdown 文档
- `.extension` 扩展配置
- `url.txt` 部署链接清单

每个方案卡会显示站点、区域数量、部署形态和各类文件数量。点击“查看方案”进入对应方案详情。

资产管理与 OBS 的区别：

- 资产管理关注仓库中有哪些可交付内容。
- OBS 管理只关注对象存储位置和发布约定。
- 当前主导航使用“资产管理”；旧 OBS 页面仍保留，但不代表完整资产中心。

## 10. 发布记录与报告

### 10.1 发布记录

发布记录读取 `release/` 目录，展示方案、版本、站点、Region、架构和状态。

页面显示“已发布”只代表本地发布目录存在对应资产，不自动等价于已经完成生产 OBS 上传或华为云市场上线。正式发布仍需要通过项目质量门禁和用户授权。

### 10.2 报告

报告页保留现有项目分析和汇总内容，用于辅助决策，不作为质量门禁或发布事实源。

## 11. 数据更新规则

### 11.1 Practice 目录变化

新增或删除 practice、Region 或部署形态后，在仓库根目录执行：

```bash
node scripts/gen-practices-index.mjs
cd web
npm run build
```

### 11.2 Skills 变化

更新 `skills-index.json` 后重新执行 `npm run build`。

### 11.3 质量结果变化

修改 Terraform、脚本、文档或规则后重新执行 `npm run build`。构建过程会调用仓库根目录 `.venv-sac` 中的测试运行器。

## 12. 当前限制

当前未实现：

- 真实任务创建、保存和恢复
- Agent 后端执行与调度
- SSE / WebSocket 实时日志
- 决策审批持久化
- 在线代码 Diff 和 AI 自动修复
- 从 Terraform Plan JSON 自动提取拓扑
- 交付包下载
- 真实 Terraform / RFS 直链生成
- 多用户、角色和权限控制

浏览器端不会提供任意 Shell 输入，也不会直接执行用户命令。未来接入后端时应继续使用工作流与参数白名单。

## 13. 常见问题

### 页面内容没有随仓库更新

静态页面仍在使用旧构建结果。重新执行 `npm run build` 并刷新浏览器。

### 浏览器无法访问 `0.0.0.0:4173`

`0.0.0.0` 是服务监听地址，不是通常使用的浏览器目标地址。请访问 `http://<服务器 IP>:4173/`，并确认网络和防火墙允许该端口。

### 方案详情返回 404

详情页只静态生成正式方案。检查该方案是否列在 `project.config.json` 的 `formal.practices` 中，并重新生成索引和构建页面。

### 质量中心显示错误或警告

展开对应实例查看检查名称和说明。错误通常会阻断正式交付；警告需要结合方案合同判断是否必须修复。

### 页面按钮没有产生真实文件

当前是静态原型。任务确认、复制方案、决策确认和自动修复尚未连接服务端任务系统。

## 14. 安全与操作边界

- 不在页面、日志或演示数据中写入 AK、SK、Token、密码或私有桶地址。
- 不通过浏览器接受或执行任意 Shell 命令。
- 不把演示状态当作真实测试、审批或发布结果。
- 未经授权不创建云资源、不上传 OBS、不发布生产链接、不执行 Git 提交或推送。
- 正式交付事实以 `project.config.json`、`practices/`、测试报告和 `release/` 为准。
