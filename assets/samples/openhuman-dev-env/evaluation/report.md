# OpenHuman 华为云解决方案实践技术评估报告

> **评估对象**：OpenHuman 桌面 AI 助手云端部署
> **目标架构**：Flexus X 实例 (2vCPU 4GB) + EIP
> **评估日期**：2026-05-21
> **评估方法**：华为云解决方案实践 8 维度技术评估框架 (参考 `reference/evaluation-methodology.md`)

---

## 一、核心结论

### 结论：❌ 不可直接部署，建议采用替代方案

**根本原因**：OpenHuman 是 **Tauri 桌面应用**（Rust + React），不是 Web 服务。它没有可对外暴露的 HTTP API Server，无法通过 RFS 部署后让用户通过浏览器访问。

| 核心矛盾 | 说明 |
|---------|------|
| OpenHuman 本质 | Tauri 桌面 App，本地 GUI 应用 |
| RFS 能力 | 云资源编排（VPC/ECS/EIP） |
| 鸿沟 | 没有"部署到云上让用户访问"这个场景 |

### 替代方案可行度

| 方案 | 可行度 | 说明 |
|------|--------|------|
| **方案 A：云端开发环境** | ✅ 可行 | ECS 上预装 Node 24+ / Rust / pnpm / Ollama，开发者 SSH 进入构建和测试 OpenHuman。**本模板采用此方案** |
| **方案 B：Ollama 推理后端** | ✅ 可行 | 在 ECS 上只安装 Ollama，作为 OpenHuman 桌面的远端推理引擎。但 2u4g 无 GPU，推理速度较慢 |
| **方案 C：远程桌面运行** | ⚠️ 勉强 | ECS 装 Xfce+VNC，远程桌面运行 OpenHuman。体验差，不推荐 |
| **方案 D：直接部署为 Web 服务** | ❌ 不可行 | OpenHuman 没有 Web Server 模式 |

### 维度 1：技术可行性 — ★★☆☆☆

| 检查项 | 结果 | 说明 |
|--------|------|------|
| OpenHuman 是否可部署为 Web 服务 | ❌ | Tauri 桌面应用，无 server 模式 |
| 是否可远程访问 | ❌ | `pnpm dev` 启动 web UI 仅供开发调试 |
| 云后端是否可自建 | ❌ | LLM/OAuth/TTS 走官方付费订阅服务 |

**关键风险**：OpenHuman 没有 "部署到云上" 这个使用场景。项目方也没有提供可自建的 server 端。

### 维度 2：部署效率 — ★★★★☆

| 阶段 | 耗时 | 说明 |
|------|------|------|
| VPC + EIP 创建 | 10-30s | RFS 标准速度 |
| Flexus X 实例创建 | 30-60s | 2u4g 轻量规格 |
| user_data 执行（总） | 5-10min | 含 apt 更新和工具安装 |
| ├─ apt update + 依赖 | 1-3min | 华为云镜像加速 |
| ├─ Node.js 24 安装 | 1-2min | nvm 方式 |
| ├─ Rust 安装 | 2-3min | rustup |
| ├─ Ollama 安装 | 30s | 官方脚本 |
| └─ pnpm install（后台） | 2-5min | 开屏后自动执行 |
| 用户可访问时间 | 5-8min | ECS 创建完成 + SSH 就绪 |

### 维度 3：运维便捷性 — ★★★★☆

| 项 | 状态 | 说明 |
|----|------|------|
| SSH 访问 | ✅ | EIP + 安全组开放 22 |
| 开发环境就绪 | ✅ | Node/Rust/pnpm/Ollama 全部预装 |
| 自愈能力 | ✅ | systemd 托管 Ollama |
| 日志 | ✅ | 统一输出到 `/var/log/setup-*.log` |
| 监控 | ❌ | 需手动对接 CES（可选） |

### 维度 4：脚本兼容性 — ★★★★★

| OS | 状态 | 说明 |
|----|------|------|
| Ubuntu 22.04 | ✅ 验证通过 | 推荐，nvm/rustup 官方支持 |
| Ubuntu 20.04 | ✅ 兼容 | npm 版本可能稍旧 |
| CentOS 7.9 | ⚠️ 需调整 | 包名差异，glibc 版本可能不足 |

脚本均使用标准安装方式（nvm / rustup / 官方脚本），跨平台兼容性好。

### 维度 5：TF 模板通用性 — ★★★★★

模板采用标准 6 文件结构（`versions.tf` / `providers.tf` / `variables.tf` / `main.tf` / `outputs.tf` / `.extension`），完全参数化，可复用为其他开发环境模板。

### 维度 6：RFS 编排稳定性 — ★★★★★

仅编排 3 种资源（VPC / EIP / Flexus X），依赖链简单，无循环依赖风险。Flexus X 实例资源充足，库存风险低。

### 维度 7：批量集群能力 — ★★★☆☆

单节点开发环境。批量部署多个环境可通过 `count` 实现，但实际场景较少。

### 维度 8：故障回滚机制 — ★★★★☆

| 故障场景 | 处理方式 |
|---------|---------|
| VPC 创建失败 | RFS 自动回滚 |
| ECS 创建失败 | RFS 自动回滚 |
| user_data 失败 | 脚本内 `set -e` 及时报错，日志持久化 |
| 应用启动失败 | 手动 SSH 排查 |

### 使用流程


## 五、结论与建议

### 方案定位

**"在华为云 Flexus X 上一键搭建 OpenHuman AI 开发环境"**

这不是一个"部署 OpenHuman 供用户访问"的方案，而是一个**为开发者准备的 OpenHuman 云端开发工作站**。用户获得的是一个预装了全部开发工具链和 Ollama 推理引擎的云服务器。

### 适用用户

- 想参与 OpenHuman 开发的贡献者
- 想测试 OpenHuman 但没有本地开发环境的开发者
- 需要云端 Ollama 推理服务做测试的用户

### 不适用用户

- 希望直接通过浏览器使用 OpenHuman 的最终用户 → ❌
- 希望在生产环境部署 OpenHuman 服务的企业 → ❌

### 推荐的三条路

```
你想要的方案                    →   ❌  OpenHuman 是桌面 App，不能
"我想部署 OpenHuman 云服务"         部署为 Web 服务

最接近的方案                    →   ✅  云端开发环境，SSH 进入后
"我想在云上开发 OpenHuman"          `pnpm dev` 即可工作

更轻量的方案                    →   ✅  只装 Ollama，供桌面端
"我只想要个云端 AI 推理"             OpenHuman 连接使用
```
