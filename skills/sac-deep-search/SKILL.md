# 深度搜索调研

仅当结论需要多个独立来源、跨维度比较或争议验证时使用，例如多产品选型、行业趋势、竞争格局或系统性证据报告。普通搜索能回答的问题，以及 `sac-business-evaluator`、`sac-technical-evaluator` 自身可完成的常规评估，不使用本 Skill。

## 研究流程

1. 明确研究问题、时间范围、比较维度和决策标准。
2. 将问题拆成少量可验证的子问题，优先检索官方文档、原始数据、论文和项目仓库。
3. 阅读关键来源正文；对影响结论的事实使用独立来源交叉验证。
4. 区分事实、来源观点、推断和未知项；记录版本与发布日期。
5. 综合差异、矛盾、限制和反例，给出与证据强度相匹配的结论。

不机械要求固定层数、固定查询次数或固定网页数量；研究深度以覆盖关键问题和完成交叉验证为准，避免为了数量引入低质量来源。

## 工具与来源

- 使用当前可用的网络搜索、网页阅读、项目资源和知识库工具，不依赖特定工具名称。
- 技术问题优先采用官方文档、源代码、发布记录和论文。
- 时效性结论必须核对最新官方来源；无法验证时明确说明不确定性。
- 尊重引用和版权限制，引用应直接支持对应结论。

## 输出边界

默认只在对话中返回中文 Markdown，包含：

- 研究范围与判断标准
- 核心发现及证据
- 关键差异、风险和未知项
- 结论或建议
- 直接来源链接

只有用户明确要求保存文件时才创建 Markdown 或 HTML。只有用户明确要求外部图表时才调用 QuickChart 等外部服务；一般关系优先用简短文字、表格或本地 Mermaid 表达。不得因问题复杂就自行写文件或发起额外外部调用。

研究结果是业务或技术评估的证据输入，不得自行启动开发、修改 Solution Practice、冻结架构或执行发布。
