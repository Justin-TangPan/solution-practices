## 写作规范

### 标题命名

- **方案名**：技术名称，如 Hermes Agent、Headroom + Claude Code
- **价值场景**：一句话价值主张，如"越用越聪明的智能体"、"更省的编程助手"
- **格式**：`{方案名}，{价值场景}`

### 内容原则

1. **简洁**：每个优势/场景 1-2 句话，不展开细节
2. **价值导向**：先说价值，再说技术
3. **客户视角**：用客户语言，不用技术术语
4. **可操作**：CTA 按钮明确，部署步骤清晰

### 禁忌

- 不要在优势卡片中写技术实现细节
- 不要在 Hero 区域写长段落
- 不要使用过于专业的术语（留给技术文档）
- 不要省略费用说明和免责声明
