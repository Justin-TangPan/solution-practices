## 1. 方案简介

### 1.1 方案组成

本方案由两个核心组件组成：

**OpenCode** 是一款开源的 AI 编程助手，运行在终端中，支持代码生成、重构、调试、审查等 AI 辅助编程能力。开发者通过自然语言与 OpenCode 交互，即可完成复杂的编程任务。OpenCode 通过 `opencode.json` 配置文件管理 API 连接和模型参数，安装简便，开箱即用。

**Headroom** 是一款开源的上下文压缩代理，部署在 OpenCode 和大模型 API 之间。它自动压缩工具输出、文件内容、对话历史和日志，在数据到达模型之前完成优化，实测压缩率 60-95%，答案质量不降。

### 1.2 一句话定位

> **一键部署的 AI 编程辅助环境，通过智能上下文压缩技术，将 Token 成本降低 60-95%，同时保持对话质量和开发体验不变。**

### 1.3 适用客户

| 客户类型 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| **Vibe Coding 开发者** | 用自然语言描述需求，AI 生成完整代码 | 降低 Token 成本，延长对话窗口 |
| **大型代码库团队** | 百万行级代码库的搜索、理解、重构 | 压缩工具输出，避免上下文溢出 |
| **DevOps / SRE 团队** | 大量日志分析、故障排查 | 压缩日志，快速定位问题 |
| **技术写作团队** | 文档生成、知识库维护 | 保持风格一致性，降低重复劳动 |
| **合规审查团队** | 金融、医疗等强监管行业内容审查 | 数据本地处理，不出域 |

## 2. 应用场景

### 场景一：Vibe Coding — 零门槛 AI 编程

**适合客户：** 非专业开发者、产品经理、设计师，或希望快速原型开发的团队

**使用方式：**
- 用自然语言描述需求："帮我写一个用户登录页面，支持邮箱和手机号"
- OpenCode 自动生成完整代码，包括前端、后端、数据库
- 通过对话迭代优化，无需手写代码

**Headroom 价值：**
- Vibe Coding 通常需要多轮对话，上下文快速增长
- Headroom 压缩历史对话，保持窗口在可控范围
- Token 成本降低 60-90%，让 Vibe Coding 真正可用

---

### 场景二：大型代码库探索与重构

**适合客户：** 管理百万行级代码库的中大型企业（50-500 人）

**使用方式：**
- 用自然语言搜索代码："找到所有调用支付接口的地方"
- OpenCode 读取多个文件，分析依赖关系，输出重构建议
- 多轮对话讨论重构方案，一次讨论完成

**Headroom 价值：**
- 代码搜索场景压缩率 92%（17,765 → 1,408 Token）
- 可同时加载多个模块文件，架构师获得全局视野
- 重构方案评审时间从 3 天缩短至 1 天

---

### 场景三：日志密集型故障排查

**适合客户：** DevOps / SRE 团队，需要 AI 协助排查生产环境问题

**使用方式：**
- 将大量日志喂给 OpenCode："分析这段日志，找出错误原因"
- OpenCode 识别异常模式，定位根因，给出修复建议
- 通过 `headroom_retrieve` 可逆取回原始日志

**Headroom 价值：**
- 日志压缩率 92%，可完整分析 10 万行日志
- 故障平均定位时间（MTTR）从 4 小时缩短至 1.5 小时
- 年减少业务损失数百万元（按每次故障影响 ¥50,000 计算）

## 3. 架构与部署

### 3.1 架构描述

本方案采用**三层架构**，在华为云 Flexus X 实例上部署完整的 AI 编程环境：

```
┌─────────────────────────────────────────────────────────────┐
│                        用户终端                               │
│  SSH Client / Terminal                                       │
└─────────────────────────┬───────────────────────────────────┘
                          │ SSH (端口 22)
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Flexus X 实例 (ECS)                                         │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  OpenCode CLI                                          │  │
│  │  opencode.json → base_url: http://localhost:8787       │  │
│  └──────────────────────┬────────────────────────────────┘  │
│                         │ HTTP (本地端口 8787)               │
│  ┌──────────────────────▼────────────────────────────────┐  │
│  │  Headroom Proxy (后台进程)                              │  │
│  │  • AST 感知压缩（Python/JS/TS/Go/Rust/Java/C++）       │  │
│  │  • 工具输出 / 文件内容 / 对话历史 / 日志压缩            │  │
│  │  压缩率：60-95%                                        │  │
│  └──────────────────────┬────────────────────────────────┘  │
│                         │ HTTPS (公网)                       │
└─────────────────────────┼───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  华为云 MaaS API (ModelArts)                                 │
│  https://api.modelarts-maas.com/anthropic                   │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  模型推理                                              │  │
│  │  • deepseek-v3.2（性价比高，日常编程）                   │  │
│  │  • claude-sonnet-4-20250514（复杂推理、架构设计）        │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

**数据流说明：**
1. 用户在终端输入问题 → OpenCode CLI 捕获
2. OpenCode 将请求发至 `http://localhost:8787`（Headroom 代理）
3. Headroom 代理对请求上下文做压缩处理（AST 解析、去重、结构化压缩）
4. 压缩后的请求通过 HTTPS 转发至华为云 MaaS API
5. MaaS 调用大模型推理，返回结果
6. Headroom 将结果返回给 OpenCode，原始数据本地保存

### 3.2 基础设施组件

| 组件 | 用途 | 配置 |
|------|------|------|
| **Flexus X 实例** | 部署 Headroom 代理和 OpenCode | x1.2u.4g，Ubuntu 24.04 |
| **弹性公网 IP** | 提供公网访问能力 | 300Mbit/s，按流量计费 |
| **安全组** | 网络访问控制 | 放行 SSH(22)、代理(8787)、ICMP |
| **VPC + 子网** | 隔离的虚拟网络 | 172.16.0.0/16 |

### 3.3 部署方式

**一键部署（推荐）：**
- 通过华为云解决方案实践或 RFS 控制台一键部署
- 部署时长：约 10 分钟
- 预估费用：约 277.60 元/月（按需计费）

**手动部署：**
- SSH 登录 ECS，执行安装脚本
- 配置 `opencode.json`，启动 OpenCode
- 适合需要自定义配置的场景

### 3.4 核心配置

OpenCode 通过 `opencode.json` 配置文件管理 API 连接。Headroom 代理通过环境变量配置上游 MaaS 接入。

**OpenCode 配置（opencode.json）：**

| 配置项 | 值 | 作用 |
|--------|-----|------|
| `base_url` | `http://localhost:8787` | OpenCode → Headroom 代理 |

**Headroom 代理配置（环境变量）：**

| 配置项 | 值 | 作用 |
|--------|-----|------|
| `ANTHROPIC_BASE_URL` | `http://localhost:8787` | 代理监听地址 |
| `ANTHROPIC_TARGET_API_URL` | `https://api.modelarts-maas.com/anthropic` | Headroom 代理 → MaaS 上游 |
| `ANTHROPIC_AUTH_TOKEN` | MaaS API Key | 认证凭据 |
| `ANTHROPIC_MODEL` | `deepseek-v3.2` | 使用的模型 |

## 4. 核心优势

### 优势一：60-95% Token 成本压缩

| 工作负载类型 | 压缩率 |
|-------------|-------|
| 代码搜索 | **92%** |
| 日志输出 | **92%** |
| 聊天历史 | **89%** |
| 文件内容 | **88%** |
| MCP 工具输出 | **86%** |
| 代码探索 | **47%** |

### 优势二：零代码改动，透明接入

- 仅需在 `opencode.json` 中将 `base_url` 指向 `http://localhost:8787`
- 开发者无感知，就像直接连接 API 一样
- 现有工作流和工具链零迁移成本

### 优势三：可逆压缩，不丢信息

- 原始数据本地保存，模型可通过 `headroom_retrieve` 按需取回
- 在节省 Token 和保证推理质量之间取得最佳平衡
- 解决了"压缩会丢信息"的核心顾虑
